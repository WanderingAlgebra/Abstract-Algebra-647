# 写作与维护指南

本文件面向后续维护者和 AI agent，不面向普通读者。README 只保留项目介绍、来源范围、编译方法和内容概览。

## 写作风格

**风格来源必须先读。** 凡是新增、改写或大幅调整 `chapters/` 中的正文，agent 在动笔前必须先阅读两份风格来源：

1. `.claude/math-latex-note-local-style/SKILL.md` — 本地 LaTeX 笔记的写作规范，包括防模板化方法论、概念讲解、定义呈现、证明处理、例题编排等具体规则。
2. `647老师风格.md` — 原始教师风格画像，用于判断讲解顺序、详略分配和条件敏感性。

若没有读过这两份文件，不应声称正文已经按老师风格或本地风格处理。若本轮只做纯机械操作（如修 typo、改 label、移动文件、不改变正文叙述），可以跳过。

## 从课堂流水稿改写为 TeX

本项目的主要工作流是：先有 Obsidian 中的原始课堂流水笔记，再由 agent 整理成 `chapters/` 中的 LaTeX 正文。

改写时应注意：

- 第一步不是润色句子，而是还原课堂逻辑。同一段课堂推进可以合并成连续叙述，真正承担转折作用的地方再分段。
- 原始稿中的数学内容要尽量保留，包括例子顺序、证明目标、老师特别提醒的条件。可以补全跳过的推导，但补全必须来自已知定理或标准事实；如果某一步需要新引理，正文中应明确说出来。
- 遇到口头化表达时，转成清楚但不过度工整的课堂书面语。改写完成后应逐段检查：自言自语式表述（”需要看一下””不妨想想”）替换为直接陈述；模糊量词（”很多””一些”）给出具体例子或删除；英文术语首次出现时给出中文对应。
- 不要把 agent 的工作说明、用户反馈或写作策略写进正文。
- 涉及定义或公理验证时，优先写具体等式、封闭性计算或有限和展开，再用一句总结性语言收束。遇到双重求和、卷积、基展开时，必须说明指标遍历什么、一项从哪里来、系数怎么乘。
- 改写完成后，回看原始稿，确认没有漏掉老师强调的点，也没有把尚未证明的说法写成已证定理。

## 思考题格式

思考题只做正文停顿，不是每节的固定配置。只有当某处确实值得读者停下来判断条件边界、尝试关键构造、识别常见误解时，才加入。每个 subsection 最多一个；没有高质量问题就不加。

```tex
\hypertarget{q:...}{}
\medskip
\noindent\textbf{思考：...}（\hyperlink{ans:...}{跳转到解答}。）
\medskip
```

对应答案统一放在本节末尾的”课堂问题参考解答”中：

```tex
\begin{exercise}\label{ex:ans-...}
\hypertarget{ans:...}{}
\textbf{问题标题}

解答正文。

\hyperlink{q:...}{返回问题}
\end{exercise}
```

## 组织约定

- 保留清晰的 `subsection` 层级，不滥用更细的小标题。
- 可以使用 `definition`、`theorem`、`exercise` 等模板内置环境。
- 不新增 `qbox`、`examplebox`、`warnbox`、`infobox` 等彩色方块。
- 数学术语前后统一。同一概念使用中文名或统一的英文名，不要交替混用（如”quiver”和”有向图”应选定一个并在首次出现时注明另一个）。
- 同一符号在不同例子中出现时，含义应一致或明确说明变更。
- 修改正文后运行两遍 `xelatex -interaction=nonstopmode main.tex`，确认目录、编号和超链接刷新正常。

## Agent 自检清单

Agent 写完或改完正文后，应自我检查涉及的证明和公式推导，不能只检查行文是否顺：

- 每一步使用的条件是否已经给出或已经证明，不能提前使用后文结论。
- 变量所属的域、环、向量空间是否清楚，标量域有没有偷换。
- 等价变形是否真的是双向等价，还是只证明了单向推出。
- 是否出现循环论证，例如用”某集合已经是域”去证明它等于要构造的最小域。
- 线性无关、生成、封闭、存在逆元分别证明了什么，不能用其中一项替代另一项。
- 结论是否说得比已证明内容更强，例如只证明了代数数构成域，就不要直接声称已经证明了代数闭包。

## 后续扩展

添加新章节时，在 `chapters/` 中新建对应文件，并在 `main.tex` 中加入：

```tex
\chapter{章节名}
\input{chapters/chXX}
```
