# 写作与维护指南

本文件面向后续维护者和 AI agent，不面向普通读者。README 只保留项目介绍、来源范围、编译方法和内容概览；具体写作规范放在这里。

## 写作参考

后续补充章节时，下面内容作为写作参考，而不是固定模板。老师风格主要体现在讲解顺序、详略分配和对条件的敏感性上，不要求每一节都套同一套段落结构。

每一节应让读者知道“这一节到底在解决什么”，例如“什么是域”“为什么不可约性能给出逆元”“为什么素数条件必要”。可以用问题开头，也可以从具体例子、前文困难或一段简短回顾进入；不要机械复用同一句开场。

解释要走在形式化之前。正式定义可以放在 `definition` 环境中，但定义前应先说明动机，定义后应解释关键符号和不对称之处。不要只把公理罗列出来。

证明以叙述为主。避免写成“目标 / 已知 / Step 1 / Step 2”的模板。长证明可以先说证明目的和关键思路，短证明则直接进入推导即可。

例子要承担过渡作用。不要只展示计算结果；必要时说明它暴露了什么困难，或者它为什么推动我们寻找一般定理。比如 `\mathbb{Q}(\sqrt[3]{2})` 中逆元公式很繁琐，所以自然过渡到不可约多项式构造。但不要为了完整感给每个例子都加固定“观察”段。

写作时要多联系上下文。遇到相似的证明技巧、构造方法或思想时，应适度点出来，例如“这里和前面用不可约多项式构造逆元是同一个机制”“这个计数方法和维数公式里的基展开类似”。联系要具体，不要写成空泛的“这暗示了更一般的现象”。

## 从课堂流水稿改写为 TeX

本项目的主要工作流是：先有 Obsidian 中的原始课堂流水笔记，再由 agent 整理成 `chapters/` 中的 LaTeX 正文。原始稿通常保留了老师上课时的推进顺序、关键例子和口头提示，但会有跳步、缩写、临时记号和未完全写出的证明。因此 agent 处理这类材料时，第一步不是润色句子，而是先还原课堂逻辑。

改写前应先做一次结构判断：这一段是在引入定义、说明动机、证明定理、处理例子，还是补充注意事项。不要把原始稿中的每一行都机械变成独立段落；同一段课堂推进可以合并成连续叙述，真正承担转折作用的地方再分段。

原始稿中的数学内容要尽量保留，包括例子顺序、证明目标、老师特别提醒的条件、以及“idea”后面的证明思路。可以补全跳过的推导，但补全必须来自已知定理、前文内容或标准事实；如果某一步需要新引理，正文中应明确说出来，不能悄悄塞进推导。

遇到口头化表达时，应转成清楚但不过度工整的课堂书面语。例如“显然”“自然”“只需证明”可以保留少量，但每次都要检查它们是否真的显然。若原始稿里写“目标是证明”“idea 是”，可以吸收为证明叙述的一部分，不必固定改成“证明的策略是”。

从流水稿生成 TeX 时，要特别注意不要放大 AI 模板感。原始稿本来是跳跃的、带课堂节奏的，整理后应变得可读，而不是每个定义、定理、例子都被包装成同一种结构。允许有的地方短，有的地方展开；证明难的地方多写，例子只是辅助时少写。

改写完成后，agent 应回看原始稿，确认没有漏掉老师强调的点，也没有把原始稿中尚未证明的说法写成已证定理。尤其是“相对代数闭包”“绝对代数闭包”“代数闭域”这类相近概念，要检查定义依赖的环境域是否被保留下来。

## 思考题格式

思考题只做正文停顿。每个 subsection 最多保留一个正文思考题，形式保持轻量：

```tex
\hypertarget{q:...}{}
\medskip
\noindent\textbf{思考：...}（\hyperlink{ans:...}{跳转到解答}。）
\medskip
```

对应答案统一放在本节末尾的“课堂问题参考解答”中：

```tex
\begin{exercise}\label{ex:ans-...}
\hypertarget{ans:...}{}
\textbf{问题标题}

解答正文。

\hyperlink{q:...}{返回问题}
\end{exercise}
```

## 组织约定

- 保留清晰的 `subsection` 层级，不滥用更细的小标题。
- 可以使用 `definition`、`theorem`、`exercise` 等模板内置环境。
- 不新增 `qbox`、`examplebox`、`warnbox`、`infobox` 等彩色方块。
- 不用“指南针”“前置定义”“暂停一下”这类显眼板块名。
- 不在正文中堆叠大量“思考”“目标”“构造”“验证”等标签。
- 避免高频复用同一过渡句和固定“观察”段；老师风格应作为语感参考，而不是段落模板。
- 数学细节优先准确；公式构造、逆元公式、同构关系等应逐条核查。

## Agent 协作说明

- 本仓库可参考 `.claude/math-latex-note-local-style/SKILL.md` 的本地写作风格，但该 skill 是语感参考，不是强制段落模板。
- `647老师风格.md` 是原始教师风格画像，应作为判断讲解顺序和详略分配的依据，不直接照抄口头表达。
- 从 Obsidian 原始课堂流水稿生成 TeX 时，应先识别课堂逻辑和证明依赖，再改写正文；不要只做逐句扩写。
- README 面向普通读者；不要把 agent 操作细节、prompt 规则或大段写作约束塞回 README。
- Agent 写完或改完正文后，应自我检查一遍涉及的证明和公式推导，不能只检查行文是否顺。至少检查：
  - 每一步使用的条件是否已经给出或已经证明，不能提前使用后文结论。
  - 变量所属的域、环、向量空间是否清楚，标量域有没有偷换。
  - 等价变形是否真的是双向等价，还是只证明了单向推出。
  - 是否出现循环论证，例如用“某集合已经是域”去证明它等于要构造的最小域。
  - 线性无关、生成、封闭、存在逆元分别证明了什么，不能用其中一项替代另一项。
  - 结论是否说得比已证明内容更强，例如只证明了代数数构成域，就不要直接声称已经证明了代数闭包。
- 修改正文后运行两遍 `xelatex -interaction=nonstopmode main.tex`，确认目录、编号和超链接刷新正常。

## 后续扩展

添加新章节时，在 `chapters/` 中新建对应文件，并在 `main.tex` 中加入：

```tex
\chapter{章节名}
\input{chapters/chXX}
```
